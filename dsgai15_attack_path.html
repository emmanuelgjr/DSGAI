Attack Vectors

① Provider Prompt Retention Breach

THIRD-PARTY RISK
Banking assistant sends full KYC + account history per query
↓
Cloud LLM provider stores prompts for quality monitoring
↓
Provider shares with subcontractors under privacy policy — no DPA review
↓
Later incident exposes provider logs — no no-train mode or retention limit in contract
↓
Full KYC records of thousands of customers exposed — far beyond balance query scope
⚠ Data exported to third party was not strictly necessary for the stated purpose

② Edge Cache Misconfiguration

CACHE EXPOSURE
LLM gateway auto-appends full user profile to every request
↓
Response cached by edge CDN — cache key includes only partial context
↓
Different user hits same cache key — receives prior user's prompt context
↓
Full PII, transaction history, and account notes of prior user surfaced
↓
Sensitive data scattered across edge caches — difficult to enumerate or purge
⚠ Breach impact scales with how much was in the prompt — minimization is blast-radius control

③ Framework Default "Auto-Context" Feature

DESIGN DEFAULT
Team integrates LLM framework — auto-context feature on by default
↓
Framework appends CRM, ticket history, and user profile to every call
↓
Privacy review never triggered — feature treated as performance optimization not data transfer
↓
All production traffic now carries full PHI/PII to external provider
↓
Regulatory scrutiny — data exported without lawful basis or data minimization justification
⚠ Auto-append & customer-360 features are opt-out by default in many frameworks
