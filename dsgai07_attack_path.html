DSGAI07 · Attack Path

Data Governance, Lifecycle & Classification

How governance failures propagate into derived artifacts and make remediation impossible

Primary Failure Flow
Unclassified Data Ingested
No PHI/PII label — passes ingestion gate unchecked
→
Derived Artifacts Created
Embeddings, fine-tune datasets, backups — no classification inherited
→
Source Record Deleted
DSR erasure request or retention schedule honored — but only at source
→
Derived Artifacts Persist
Embeddings remain live · backups retain raw content · no lineage to trace
→
💥 Erasure Violated
Deleted data re-enters retrieval surface — cannot be scoped or remediated

Stage 2
Lifecycle Obligations Not Applied to Derived Artifacts
DSR erasure request received — source record deleted ✓
↓
Embedding persists in vector store — not linked to source
↓
Fine-tuning run not inventoried — no source linkage
↓
Nightly backup retains raw content — not purged on schedule
↓
Re-indexing job ingests backup — deleted record re-enters live retrieval
⚠ Org believes erasure obligation honored — it has not been
→
Stage 3
Lineage Gaps Make Remediation Impossible
Regulator requests proof of consent for fine-tuning dataset
↓
No artifact linking training run to dataset version
↓
No consent records mapping to specific training records
↓
Cannot scope blast radius of compromised data source
↓
Cannot prove compliance · Cannot execute targeted remediation · Cannot honor erasure at weight level
⚠ Machine unlearning cannot be scoped without data-to-model lineage
