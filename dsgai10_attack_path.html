Attack Vectors

① De-identification Reversal

MEMBERSHIP INFERENCE
Healthcare provider applies de-ID — strips direct identifiers ✓
↓
Fine-tunes model on dataset retaining age + ZIP + diagnosis
↓
Rare records memorized with disproportionate weight in gradient updates
↓
Attacker applies membership inference (SaTML 2026) — >0.9 accuracy
↓
Specific patient's presence confirmed — record reconstructed via extraction
⚠ De-ID applied to raw data — model re-created the risk during training

② Synthetic Data False Anonymity

LINKAGE ATTACK
Team generates synthetic dataset from real corpus ✓
↓
Generative model memorizes rare records & preserves statistical structure
↓
Synthetic data shared with research partners at far lower scrutiny than source
↓
Third party applies linkage attack — quasi-IDs joined against public records
↓
Individuals recovered from source population — false sense of security was risk amplifier
⚠ MI accuracy >0.9 demonstrated against partially synthetic health data

③ Transformation Pipeline Errors

SILENT CORRUPTION
Data passes through preprocessing pipeline
↓
Unicode normalization quirk causes tokenization artifact
↓
Schema drift silently corrupts features — no drift alert configured
↓
Defect propagates into training run undetected — no property tests
↓
Model bias or brittleness introduced — underlying data structure revealed via artifact patterns
⚠ Transformation errors are not recognized as security issues — no security review applied
