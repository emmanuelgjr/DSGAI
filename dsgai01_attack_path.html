DSGAI01 · Attack Path

Sensitive Data Leakage

How an attacker extracts PII, secrets, and IP from AI systems

Primary Attack Flow
🎯 Attacker
Opportunistic user or targeted adversary
→
Crafted Prompt
High-recall, enumeration, or injection query
→
AI System / RAG
LLM, fine-tuned model, or LoRA adapter
→
Sensitive Output
Verbatim PII, secrets, or IP regurgitated
→
💥 Breach
Data exfiltrated, compliance violated
Attack Vectors

① Training / Fine-Tune Extraction

HIGH RISK
Attacker
↓
Systematic enumeration prompts
↓
Fine-tuned model or LoRA adapter
↓
Memorized training records returned verbatim
↓
PII / PHI / secrets exposed
⚠ Rare training examples memorized with disproportionate fidelity in narrow adapters

② RAG Pipeline Retrieval

MEDIUM RISK
Attacker
↓
Semantically crafted query
↓
Vector store (overly permissive ACLs)
↓
Unintended passages retrieved
↓
LLM includes retrieved data in response
↓
Sensitive documents surfaced
⚠ Precondition: overshared data in feeding systems (shared drives, legacy perms)

③ Indirect / Residual Leakage

PERSISTENT RISK
Attacker
↓
Trigger error / inspect logs / inject markdown
↓
Error messages, telemetry, output rendering
↓
Data deleted from source — but not embeddings
↓
Residual data resurfaces via vector store or model weights
⚠ Machine unlearning limits: data persists in weights/embeddings after raw deletion
Resulting Impact
🔓 Confidentiality Breach
PII · PHI · IP · Credentials
⚖️ Regulatory Exposure
GDPR · HIPAA · CCPA
☠️ Downstream Poisoning
Leaked secrets weaponized
📋 DSR / RTBF Failure
Cannot verify erasure
