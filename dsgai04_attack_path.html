DSGAI04 · Attack Path

Data, Model & Artifact Poisoning

A three-stage continuous lifecycle from supply chain compromise to poisoned production model

Primary Attack Flow
🎯 Attacker
Controls upstream source or supply chain node
→
Injects Malicious Artifact
Poisoned dataset, model file, package, or script
→
Enters Pipeline Undetected
Unsigned artifact passes superficial review
→
Poisons Training / RAG
Backdoors, bias, or privacy control removal baked into weights
→
💥 Production Compromise
Backdoors activate; credentials exfiltrated; data extracted
Three-Stage Attack Lifecycle
Stage 1
Supply Chain Compromise
Attacker targets upstream source
↓
Typosquatted package / malicious model hub file / compromised token
↓
ML pipeline fetches artifact
↓
Model __init__ executes on load — reads env vars, exfiltrates credentials

→
Stage 2
Artifact Tampering
Attacker gains write access to pipeline
↓
Modifies preprocessing script — disables DP-SGD noise injection
↓
Change appears as "efficiency improvement" — passes code review
↓
Model now memorizes sensitive training records DP was designed to protect
⚠ Unsigned artifacts flow through promotion gates — no integrity check
→
Stage 3
Poisoning at Training / RAG
250 crafted samples (0.00016% of tokens)
↓
Backdoor triggers or adversarial embeddings reach training / RAG index
↓
Passes standard QA — trigger activates only on specific input
↓
Production model: backdoored behavior / adversary content preferentially retrieved
⚠ Anthropic research: 250 samples = measurable behavioral impact
