Failure Vectors

① Vector DB Saturation

AVAILABILITY ATTACK
Agentic workflow issues high-recall retrieval queries in a loop
↓
Or: hostile actor sends adversarial high-cardinality queries
↓
Vector similarity search scales with index size — no circuit breaker
↓
Latency degrades — 5xx errors propagate into RAG quality
↓
All applications sharing retrieval tier affected — blast radius is shared infrastructure-wide
⚠ Unlike SQL, vector search has no bounded query plan — complexity scales unbounded

② Stale Replica Serves Deleted Records

DSR VIOLATION
Customer submits GDPR Art. 17 erasure request ✓
↓
Record deleted from primary index ✓
↓
Primary fails over to replica — 18h behind — erasure not propagated
↓
Deleted record retrieved and surfaced in RAG responses for hours
↓
Compliance team discovers exposure in audit 2 weeks later — DSR obligation violated
⚠ Stale replicas create compliance exposure independent of primary system correctness

③ Corrupted Artifact — Unvalidated Restore

RECOVERY FAILURE
Ransomware or targeted corruption hits model registry or embedding store
↓
Team restores from backup — row counts match ✓
↓
No semantic validation — nearest-neighbor results not tested against known-good pairs
↓
Restored index produces semantically incorrect retrieval — model checkpoint outputs degraded
↓
Production serving corrupted artifacts — silent misinformation at scale until discovered
⚠ Generic DB restore drills do not validate AI artifact semantic correctness
